# 【模块 1】首页启动选择页（品牌 + 交互模式）精细化重构

> **核心目标**: 新增主品牌+竞品比价体系，彻底解决原有多品牌权重一致、无法区分核心观测对象与对标对象的根本缺陷

---

## 当前原生缺陷（问题诊断）

1. 模式 C 支持多品牌博弈，但原有逻辑：所有模式默认品牌单选，与多品牌竞品博弈逻辑冲突；
2. 用户无法直观理解 A/B/C 玩法差异，首次上手门槛极高，无分层角色认知；
3. 无参数前置提示：用户不清楚选定品牌后加载的AI能力、商品池、博弈基线；
4. 无严格校验拦截：存在无效组合（模式C仅选1个品牌，无法形成竞品竞争闭环）；
5. 无主品牌+竞品拆分体系，所有品牌权重一致，无法实现「重点观测主品牌、竞品施压对标」的商业仿真核心场景。

---

## 1.1 全新品牌体系：主品牌（核心观测）+ 竞品池（对标博弈）

### 可选品牌池（全局固定）

| 品牌 | 赛道 | 核心定位 | 谈判风格 | 适用竞品对标场景 |
|------|------|----------|----------|-----------------|
| 🐶 京东 | 电商平台 | 品质+物流双驱动 | reliable（稳定可靠） | 全品类价格对标、配送时效竞争 |
| 💧 脉动 | 功能饮料 | 多口味+年轻化 | innovator（创新体验） | 消费品口味/包装差异化竞争 |
| 📷 佳能 | 影像设备 | 专业品质+镜头生态 | premium（品质优先） | 高端品类技术参数对标 |
| 🏪 沃尔玛 | 线下商超 | 每日低价+规模效应 | aggressive（激进价格） | 零售日用品价格战 |

> **全局最多4个品牌参与博弈**，每场博弈中主品牌1个 + 竞品1-3个

### 核心定义

| 角色类型 | 定义 | 数量限制 | 特权 |
|----------|------|----------|------|
| **主品牌** | 本次谈判核心观测、复盘、分析的目标品牌 | 全局最多1个 | 独立参数面板、独立复盘分析、全维度数据追踪 |
| **竞品池** | 参与同台比价、竞争博弈的对标品牌 | 1-3个，不可与主品牌重复 | 通用竞品参数控制、横向对标数据展示 |

### 品牌选择数据结构（新增）

```javascript
// 品牌选择状态模型
{
  primaryBrand: {
    id: 'walmart',        // 品牌唯一标识
    name: '沃尔玛',
    role: 'primary',      // primary | competitor
    weight: 1.0           // 观测权重（复盘时主品牌数据加权）
  },
  competitorPool: [
    { id: 'jd', name: '京东', role: 'competitor', aggressionLevel: 'medium' },
    { id: 'mizone', name: '脉动', role: 'competitor', aggressionLevel: 'high' },
  ],
  totalParticipants: 3,   // 总参与品牌数 = 1 + competitorPool.length
  crossMarket: false      // 是否跨赛道博弈（如京东vs沃尔玛）
}
```

## 1.2 三模式联动品牌选择规则（强约束、无逻辑冲突）

### 模式A：你 = 消费者AI VS 单个品牌AI

| 配置项 | 规则 | 说明 |
|--------|------|------|
| 主品牌 | 单选（必填） | 消费者的谈判对手 |
| 竞品池 | 可选开启/关闭 | 0-3个任意选择 |
| 开启竞品 | 消费者AI持有竞品报价筹码，可主动施压主品牌 | 比价筹码模式 |
| 关闭竞品 | 纯一对一封闭谈判 | 经典议价模式 |
| 总品牌数量 | 1（主）+ 0/1/2/3（竞品） | 自由组合，无强制下限 |
| 用户交互 | 手动输入谈判话术 | 自主制定议价策略 |

### 模式B：你 = 品牌AI（主品牌）VS 消费者AI

| 配置项 | 规则 | 说明 |
|--------|------|------|
| 主品牌 | 单选（必填） | 用户扮演的品牌主体 |
| 竞品池 | 可选1-3个 | 模拟消费者携带多家竞品报价上门谈判 |
| 核心场景 | 用户需应对竞品低价、权益冲击 | 守住自身成交与利润 |
| 总品牌数量 | 1（主）+ 1-3（竞品） | 至少1个竞品才能形成压力场景 |
| 用户交互 | 代表主品牌出价、策略选择 | 销售/商务谈判训练 |

### 模式C：全自动旁观 | 消费者AI ↔ 多品牌AI全自动博弈

| 配置项 | 规则 | 说明 |
|--------|------|------|
| 优先级 | ⭐ 核心优先级最高 | 唯一支持全自动多品牌比价博弈 |
| 主品牌 | 单选（必填） | 重点观测对象 |
| 竞品池 | **必填至少1个** | 总参与品牌≥2，否则下一步置灰 |
| 强制竞争闭环 | 消费者AI + 主品牌AI + 多竞品AI | 完整三方博弈 |
| 用户交互 | 播放控制（启动/暂停/单步/快进） | 纯观测模式 |

### 各模式品牌组合有效性矩阵（新增）

| 模式 | 主品牌 | 竞品数=0 | 竞品数=1 | 竞品数=2 | 竞品数=3 |
|------|--------|----------|----------|----------|----------|
| A | 1 | ✅ 有效 | ✅ 有效 | ✅ 有效 | ✅ 有效 |
| B | 1 | ❌ 无效（无竞争压力） | ✅ 有效 | ✅ 有效 | ✅ 有效 |
| C | 1 | ❌ 无效（无法形成竞争） | ✅ 有效 | ✅ 有效 | ✅ 有效 |

## 1.3 模式切换联动机制（防出错）

### 切换规则

```
切换任意模式时：
  1. 自动清空原有主品牌、竞品池选择
  2. 重置UI状态为初始态
  3. 自动切换选择组件形态：
     - A/B模式 → 轻量化选择（单区域+可选竞品）
     - C模式 → 强制双区域选择（主品牌区+竞品区，竞品区必填≥1）
  4. 实时校验：
     - 禁止主品牌重复入选竞品池（实时灰化已选品牌）
     - 禁止超量选择（竞品上限3个，达到上限后其余灰化）
     - 模式C竞品不足时【进入战场】按钮置灰+红色提示文案
```

### 校验状态机（新增）

```
[用户选择模式] → [清空品牌选择] → [渲染对应UI]
                                      ↓
[用户选择主品牌] → [更新竞品池可选列表（排除主品牌）]
                                      ↓
[用户勾选竞品] → [实时检查：总数≤4? 主品牌∉竞品池?]
                                      ↓
               → [通过] → [启用进入按钮]
               → [失败] → [按钮置灰 + toast错误提示]
```

# 【模块 2】场景选择与博弈底层参数配置面板（新增比价专项参数）

> **核心目标**: 从"仅有场景名称无变量"升级为"场景×参数×比价专项变量"三位一体可配置博弈环境

---

## 当前原生缺陷

仅有场景名称，无规则、无可调节变量；AI行为固定，同一场景博弈结果同质化严重；无任何比价、竞品对抗参数，无法模拟真实市场差异化竞争。

## 2.1 场景池基础设计（保留原有+补强场景说明）

### 场景数据结构升级

每条场景卡片固定信息结构：**场景名称 + icon + 博弈目标 + 双方基线 + 比价适配说明 + 复杂度标签**

#### 场景1：预算议价 💰

| 维度 | 内容 |
|------|------|
| 消费者目标 | 依托竞品比价，尽可能压低主品牌成交价 |
| 主品牌目标 | 不低于内部底价，优先成交、规避竞品分流 |
| 竞品目标 | 释放价格优势，抢夺消费者决策偏好 |
| 终止基线 | 僵局达回合上限则谈判破裂 |
| 比价适配 | ✅ 核心比价场景 — 竞品报价直接作为消费者施压筹码 |
| 复杂度 | ⭐⭐⭐ 高（多轮价格拉锯） |

#### 场景2：补货协商 🔄

| 维度 | 内容 |
|------|------|
| 消费者目标 | 以最优批量价格+配送周期锁定长期供应 |
| 主品牌目标 | 争取订阅锁定、提升客单价、降低履约成本 |
| 竞品目标 | 以更低订阅价或更灵活周期争夺长期合约 |
| 终止基线 | 双方对批量折扣率达成一致或一方退出 |
| 比价适配 | ✅ 竞品订阅价/批量折扣率作为对标基准 |
| 复杂度 | ⭐⭐ 中（参数多但对抗性较弱） |

#### 场景3：售后沟通 🛡️

| 维度 | 内容 |
|------|------|
| 消费者目标 | 获得最大程度售后补偿（退款/换货/维修+赠品） |
| 主品牌目标 | 最小化售后成本，同时维护客户关系防流失 |
| 竞品目标 | 借售后事件展示更优售后政策，吸引品牌转换 |
| 终止基线 | 售后方案达成一致或消费者决定更换品牌 |
| 比价适配 | ✅ 竞品售后政策（保修期/退换条件/补偿标准）横向对比 |
| 复杂度 | ⭐⭐ 中（情感因素+规则约束） |

#### 场景4：健康商品匹配 🏥

| 维度 | 内容 |
|------|------|
| 消费者目标 | 在过敏原/健康约束下找到最优性价比商品组合 |
| 主品牌目标 | 证明自身产品在安全+品质维度不可替代 |
| 竞品目标 | 以相似品质+更低价格或更全品类切入 |
| 终止基线 | 找到满足所有健康约束的商品或确认无货 |
| 比价适配 | ✅ 同品类安全商品横向参数对比（非纯价格战） |
| 复杂度 | ⭐⭐⭐ 高（多维约束+安全红线） |

### 场景选择交互（新增细节）

- 场景卡片hover展示完整博弈目标摘要
- 选中场景后，右侧预览该场景的默认参数配置
- 场景卡片右上角显示「比价适配」标签（绿色=适配，灰色=不适配）

## 2.2 面板拆分：基础区 + 高级参数折叠面板（新增【比价专项参数】独立分区）

### 设计原则

- 默认收起高级参数，新手无干扰
- 进阶用户可全开自定义
- 所有参数主品牌、竞品、消费者**独立可控**
- 参数修改实时预览（滑块旁显示数值+影响描述）

### 2.2.1 消费者AI基础可调参数（原有保留）

| 参数名称 | 控件类型 | 取值范围 | 默认值 | 说明 |
|----------|----------|----------|--------|------|
| 心理预算上限 | 数字输入+滑块 | ¥10 - ¥100,000 | 场景默认 | 消费者愿意支付的最高价格 |
| 最低预期价位 | 数字输入+滑块 | ¥0 - 心理预算上限 | 预算×0.6 | 消费者期望的最低成交价 |
| 谈判耐心阈值 | 滑块 | 1-10轮 | 5轮 | 超过此轮数消费者倾向妥协或放弃 |
| 竞品敏感度 | 滑块 | 0-100% | 60% | 消费者对竞品价差的敏感程度 |
| 谈判风格 | 下拉选择 | 激进/均衡/保守 | 均衡 | 影响让步幅度和施压强度 |
| 是否可放弃采购 | 开关 | true/false | true | 关闭后消费者必须成交（模拟刚需） |

### 2.2.2 主品牌AI独立可调参数（原有保留+补强）

| 参数名称 | 控件类型 | 取值范围 | 默认值 | 说明 |
|----------|----------|----------|--------|------|
| 最低可接受底价 | 数字输入 | ¥0 - 市场价×1.5 | 成本价×1.1 | 跌破此价品牌宁可不成交 |
| 单次最大让步幅度 | 滑块 | 1%-30% | 10% | 单轮最多降价百分比 |
| 促销/售后筹码权限 | 多选 | 满减/赠品/延保/免运 | 满减+免运 | 品牌本轮可用的非价格筹码 |
| 竞品应对强度 | 滑块 | 1-10 | 5 | 面对竞品低价时的反击力度 |
| 成交优先级 | 单选 | 利润优先/成交优先 | 成交优先 | 决定让步策略的核心导向 |
| **品牌人格一致性**（新增） | 滑块 | 0-100% | 80% | 越高越坚守品牌定位，越低越灵活变通 |

### 2.2.3 竞品AI通用独立参数（全新新增）⭐

| 参数名称 | 控件类型 | 取值范围 | 默认值 | 说明 |
|----------|----------|----------|--------|------|
| 竞品底价区间 | 双滑块 | ¥0 - 市场价 | 市场价×0.7-0.9 | 竞品最低成交红线（独立于主品牌） |
| 竞争激进度 | 滑块 | 1-10 | 6 | 高=主动降价抢单，中=被动应对，低=坚守定价 |
| 权益释放阈值 | 滑块 | 0-50% | 15% | 价差达到此百分比后，启用赠品/售后等差异化筹码 |
| 抢单优先级 | 单选 | 利润优先/成交优先/平衡 | 平衡 | 是否愿意牺牲利润换取成交 |
| **跟价响应速度**（新增） | 滑块 | 1-10 | 5 | 竞品感知主品牌降价后，多快跟进降价 |
| **差异化强调度**（新增） | 滑块 | 0-100% | 50% | 竞品强调自身差异化优势（非价格维度）的程度 |

> **注意**: 当竞品池有多个品牌时，每个竞品的参数独立设置，支持「全部竞品统一配置」开关

### 2.2.4 全局比价专项参数（全新核心新增）⭐

| 参数名称 | 控件类型 | 取值范围 | 默认值 | 说明 |
|----------|----------|----------|--------|------|
| 竞品信息透明度 | 单选 | 全开/半透明/封闭 | 全开 | 模拟不同市场信息环境 |
| 消费者比价频率 | 单选 | 每回合/僵局阶段/仅最后阶段 | 每回合 | 控制消费者何时启用竞品施压 |

| 竞品互感知开关 | 开关 | 开启/关闭 | 关闭 | 开启=竞品之间可互相感知报价；关闭=仅消费者掌握全量 |

### 信息透明度模式说明（新增详解）

| 模式 | 消费者可见 | 主品牌可见 | 竞品可见 | 仿真场景 |
|------|-----------|-----------|----------|----------|

## 2.4 边界控制（补强比价参数校验）

### 校验规则清单

## 3.8 僵局处理机制（适配比价场景）

### 僵局类型定义（新增分类）

| 僵局类型 | 判定条件 | 自动化处理 |
|----------|----------|------------|
| 价格僵局 | 双方连续2轮报价不变 | 消费者发起最后通牒 |
| 比价僵局 ⭐ | 存在明显竞品价差（>5%）但双方持续不让步 | 系统判定【比价僵局】，可自动触发AI策略调整或标记谈判破裂 |
| 信息僵局 | 封闭询价模式下消费者无法获取足够比价信息 | 消费者调整策略（降低比价频率） |
| 价值僵局 | 价格接近但非价格维度无法达成一致 | 系统提示双方关注权益/服务差异化 |

### 僵局自动处理流程（新增）

```
[检测到僵局] → [判断僵局类型]
                  ↓
    ┌─────────────┼─────────────┐
    ↓             ↓             ↓
 价格僵局     比价僵局⭐    信息/价值僵局
    ↓             ↓             ↓
 最后通牒    自动策略调整    维度切换提示
    ↓             ↓             ↓
 破裂/成交   继续/破裂/成交   继续/破裂
```

### 比价僵局特殊处理（新增）

当判定为比价僵局时：
1. 系统自动在对话中插入系统消息：「⚠️ 检测到比价僵局 — 竞品京东报价¥228已连续2轮低于主品牌沃尔玛¥235，价差3.0%」
2. 触发主品牌AI策略强制重评估
3. 消费者AI比价敏感度临时+20%
4. 如再持续1轮无进展 → 判定谈判破裂（消费者转向竞品）

## 6.2 品牌选择校验

| 校验规则 | 检查时机 | 违规处理 |
|----------|----------|----------|
| 禁止主品牌与竞品重复 | 用户选择竞品时实时检查 | 已选为主品牌的card灰化+不可勾选 |

## 6.3 AI话术合规校验（杜绝AI无效比价话术）⭐

| 校验规则 | 违规示例 | 处理方式 |
|----------|----------|----------|
| 禁止无依据随意提及竞品价格 | "京东只要200块"（实际京东报价¥228） | AI话术输出前校验：提及的竞品价格必须在±5%误差内匹配实时数据 |
| 所有比价话术必须匹配看板实时数据 | 引用过时的竞品报价 | 每轮AI决策时刷新竞品数据源 |
| 封闭询价模式下禁止引用未被消费者告知的竞品信息 | 主品牌主动提及"我知道京东卖228" | 信息权限校验：主品牌只能引用消费者已告知的信息 |
| 竞品互感知关闭时禁止引用其他竞品报价 | 京东说"我比脉动便宜¥14" | 信息权限校验：竞品间信息隔离 |

### 话术合规检查流程（新增）

```
[AI生成话术]
      ↓
[提取话术中的价格引用] → 正则匹配：¥\d+ / \d+元 / 便宜¥?\d+
      ↓
[与实时比价数据比对]
      ↓
┌─ 匹配通过（误差≤5%）→ 允许输出
└─ 匹配失败 → 自动修正价格数据 → 重新生成话术 → 重试上限3次 → 仍失败则标记⚠️
```

## 6.4 底价隔离规则

| 规则 | 详细说明 |
|------|----------|
| 内部决策底价仅后台/结构化/复盘可见 | 对话区域的通俗话术层**永不泄露**底价数字 |
| 自然对话中可用模糊表述 | "这已经接近我们的成本了"（✅ 允许） |
| 禁止在自然对话中给出精确底价 | "我们的底价是200"（❌ 禁止，触发合规校验） |
| 公开报价和底价在数据结构中严格分离 | `publicPrice`和`internalPrice`分字段存储 |
| 比价看板中底价显示🔒标记 | 视觉上明确区分公开和内部数据 |

## 6.6 网络异常与降级策略（新增）

| 异常场景 | 检测方式 | 降级方案 |
|----------|----------|----------|
| LLM API不可用 | fetch超时/错误响应 | 切换至内置模拟引擎，toast提示"API不可用，已切换至模拟模式" |
| API速率限制 | HTTP 429 | 自动重试3次（指数退避），仍失败则切换模拟引擎 |
| API返回非预期格式 | JSON解析失败 | 尝试正则提取+fallback默认值，记录异常日志 |
| 浏览器不支持IndexedDB | try-catch检测 | 降级为localStorage存储（容量受限） |
| localStorage已满 | try-catch写入 | 清理旧自动存档+提示用户手动清理 |

# 【模块 8】仿真逻辑核心优化（彻底解决AI信息隔离、无竞争感核心痛点）

## 原有致命痛点

多品牌AI各自独立对话、互相隔离、无法感知竞品报价，无真实市场竞争，博弈虚假、同质化严重。

---

## 8.1 全新三层信息流转机制（真实市场仿真）

### 第一层：消费者AI（全量信息持有者）

收集主品牌+所有竞品实时报价、权益、让步条件，拥有完整比价视图，每轮可自主择优施压、切换品牌。

### 第二层：主品牌AI（半信息接收者）

可接收消费者传递的所有竞品报价与竞争条件，实时更新自身防御、调价、权益策略。

### 第三层：竞品AI（可控信息感知）

默认仅感知主品牌报价（消费者传导），无法私自窥探其他竞品；可通过参数开关开启「竞品互感知」，模拟完全透明市场。

---

## 8.2 竞争博弈联动规则（核心仿真灵魂）

1. 任意品牌更新报价后，消费者AI立即同步全市场信息
2. 主品牌必须根据竞品优势动态调整谈判策略，不得一成不变
3. 竞品AI主动发起竞争扰动，形成「降价-对标-再让步」的真实商业拉锯
4. 杜绝独立对话、无效闲聊、无变动重复话术，所有行为均由比价、竞争、诉求驱动

---

## 8.3 最终仿真效果闭环

实现真实市场复刻：消费者比价择优、主品牌防守应对、竞品抢单冲击，三方动态博弈，每一次模拟结果完全不重复。

---

## 8.4 信息流状态机定义（新增）

```
                    ┌──────────────┐
     ┌─────────────│ 消费者AI 👑  │─────────────┐
     │ (全量信息)  │ 持有所有报价  │ (全量信息)  │
     ▼             └──────────────┘             ▼
┌──────────┐                            ┌──────────┐
│ 主品牌AI │← 消费者告知竞品报价 ──────→│ 竞品AI A │
│  🏪      │                             │  🐶      │
└──────────┘                            └──────────┘
     ▲                                       │
     │    ┌──────────────────────┐           │
     └────│ 竞品互感知开关=ON时  │───────────┘
          │ 竞品间可互相感知报价  │
          └──────────────────────┘
```

## 8.5 AI决策Prompt工程规范（新增）

### 消费者AI Prompt关键约束
- 必须基于实时比价数据做决策，不得凭空编造价格
- 每轮决策输出结构化JSON：`{decision, targetBrand, priceRefs[], strategy, confidence}`
- `priceRefs`数组必含所有已知品牌的`{brandId, publicPrice, round}`

### 主品牌AI Prompt关键约束
- 必须回应消费者引用的竞品信息（不可无视）
- 策略输出结构化JSON：`{newPrice, concessions[], defenseStrategy, competitorResponse}`
- `competitorResponse`字段必填，说明如何应对竞品威胁

### 竞品AI Prompt关键约束
- 基于自身底价+竞争激进度参数生成报价
- 开启互感知时，参考其他竞品报价调整策略
- 输出结构化JSON：`{newPrice, concessions[], attackStrategy}`

## 8.6 仿真随机性保证机制（新增）

| 机制 | 说明 |
|------|------|
| 随机种子 | 每场博弈生成唯一seed，控制AI决策的随机扰动 |
| 温度参数 | 消费者/主品牌/竞品各有独立temperature（0.3-0.9），越高越不可预测 |
| 策略扰动 | 每轮在基础策略上叠加±5%随机偏差，防止机械重复 |
| 初始条件敏感 | 初始报价微小差异可导致完全不同的博弈结果 |

---

## 8.7 仿真质量评估指标（新增）

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 结果多样性 | 同一配置×10场，成交品牌分布不集中（单一品牌≤50%） | 统计10场博弈成交品牌分布 |
| 策略丰富度 | 平均每场博弈出现≥3种不同策略类型 | 策略标签去重计数 |
| 信息利用率 | 消费者AI在≥80%的回合中引用了竞品信息 | 日志分析 |
| 竞争活跃度 | ≥60%的回合发生了报价变动 | 报价变动回合数/总回合数 |

## 8.8 技术实现要点（新增）

| 要点 | 方案 | 优先级 |
|------|------|--------|
| 信息流控制 | InfoChannel类，管理品牌间信息可见性矩阵 | P0 |
| Prompt构建 | PromptBuilder类，根据角色+信息权限+比价参数动态拼接prompt | P0 |
| 策略输出解析 | StrategyParser，JSON提取+fallback+校验 | P0 |
| 随机性控制 | SeededRandom类，seed→噪声序列，保证可复现 | P1 |

## 8.9 验收checklist（新增）

- [ ] 消费者AI可获取所有品牌报价（全量信息验证）
- [ ] 主品牌AI只能获取消费者告知的竞品信息（半信息验证）
- [ ] 竞品互感知关闭时竞品间无法获取对方报价
- [ ] 竞品互感知开启时竞品间可互相参考报价
- [ ] 每轮必有报价变化或策略调整（无无效轮次）
- [ ] 同配置×10场博弈结果不完全重复
- [ ] AI话术不出现无依据的竞品价格引用
- [ ] 仿真质量指标达标（结果多样性/策略丰富度/信息利用率/竞争活跃度）
